# 考研小汤包公众号封面设计系统（Cover Design System）

> Build a consistent, high-click, AI-friendly WeChat Cover Design
> System.

一个专为微信公众号打造的 AI 封面设计系统。

它不是一个 Prompt，也不是一个模板，而是一套可长期维护、持续迭代的
**Design System**。

------------------------------------------------------------------------

# 为什么要做这个项目？

很多 AI 都可以生成公众号封面，但普遍存在以下问题：

-   每次生成的风格都不一致
-   需要重复编写 Prompt
-   画面元素过多，缺少留白
-   没有品牌一致性
-   信息层级混乱，点击率不高

本项目希望建立一套统一的设计规范，让任何 AI
或设计师都能够生成一致风格的公众号封面。

------------------------------------------------------------------------

# 设计理念

> 好的公众号封面，不是展示设计能力，而是帮助读者一眼知道文章讲什么。

坚持：

-   Less Decoration. More Information.
-   Typography First
-   Information First
-   Brand Consistency

# 使用方法

## Step 1：让 AI 学习设计系统

将《考研小汤包公众号封面设计系统（Cover Design System）.md》发送给
AI，并说明：

> 请先完整阅读这份设计系统，后续所有公众号封面均严格遵循其中的规范。

支持但不限于：

-   ChatGPT
-   Claude
-   Gemini
-   Codex

------------------------------------------------------------------------

## Step 2：发送公众号文章

推荐发送完整内容，而不仅仅是标题。

支持格式：

-   Markdown
-   Word
-   PDF
-   公众号正文

如果没有文章内容，AI 应主动提醒用户提供正文，而不是直接生成封面。

------------------------------------------------------------------------

## Step 3：AI 工作流程

``` text
阅读 Design System
        ↓
阅读公众号文章
        ↓
理解文章核心内容
        ↓
提炼封面标题
        ↓
生成公众号封面
        ↓
Checklist 自检
        ↓
输出最终封面
```

# Examples（优秀案例）

每当生成满意的封面，请收录到：

``` text
examples/
```

建议文件命名：

``` text
001-胡寿松课后题划重点.png
002-奈奎斯特稳定判据.png
003-频域分析总结.png
```

每个案例建议记录：

-   文章主题
-   最终封面
-   为什么优秀
-   哪些设计值得保留

------------------------------------------------------------------------

# Bad Examples（反面案例）

建议长期收录：

-   元素过多
-   标题太小
-   没有视觉中心
-   信息堆砌
-   装饰喧宾夺主

这些案例可以帮助 AI 避免重复犯错。
