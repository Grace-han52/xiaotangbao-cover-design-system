# 公众号封面设计系统（Cover Design System）

**Version:** v1.0

## 项目目标

建立一套统一的公众号封面设计系统，使任何
AI（ChatGPT、Claude、Gemini、Codex
等）或设计师都能够生成风格一致的公众号封面。

核心目标：

> **让读者在 0.5 秒内知道文章讲什么，并愿意点开。**

------------------------------------------------------------------------

# AI 工作流程（Workflow）

## Step 1：先阅读本规范

所有设计必须遵循本规范，不允许自由发挥新的视觉风格。

## Step 2：确认是否收到文章内容

如果用户没有提供文章正文或 Markdown，不要直接生成封面。

应主动提醒：

> 请发送公众号文章正文（或
> Markdown），我会先阅读文章，再提炼封面文案，最后根据本规范生成封面。

## Step 3：阅读文章

理解：

-   本文核心观点
-   用户最关心的信息
-   最值得点击的内容

不要机械照搬文章标题。

## Step 4：提炼封面文案

提炼：

-   辅助标题（可选）
-   书名/课程名（可选）
-   主标题（视觉中心）

主标题必须一眼说明文章价值。

## Step 5：生成封面

按照本规范生成公众号横版封面。

## Step 6：Checklist

-   第一眼能否知道文章讲什么？
-   是否只有一个视觉中心？
-   是否还能删除元素？
-   是否保持品牌一致？

如有任一项不满足，请重新设计。

------------------------------------------------------------------------

# 品牌关键词

-   Minimal
-   Professional
-   Modern
-   Claude Code 风格
-   Typography First
-   留白
-   信息优先

参考风格：

-   Claude Code
-   Linear
-   Notion
-   Apple
-   Vercel

------------------------------------------------------------------------

# 排版规范

三层信息：

1.  顶部辅助信息（最小）
2.  课程/书名（中等）
3.  主标题（最大，视觉中心）

示例：

控制考研必看

胡寿松《自动控制原理》

课后题划重点

------------------------------------------------------------------------

# 配色

背景：

-   暖白
-   米白
-   浅灰白

文字：

-   深黑
-   深灰

强调色：

-   Claude Orange（约 #FF6B35）

颜色不超过三种。

------------------------------------------------------------------------

# 留白

大量留白。

宁可空，也不要满。

------------------------------------------------------------------------

# 装饰规范

允许：

-   //
-   ---
-   细线
-   点阵
-   小圆点

装饰面积不超过画面 10%。

------------------------------------------------------------------------

# 禁止出现

-   书本
-   铅笔
-   人物
-   教室
-   大量英文
-   大括号
-   红色营销按钮
-   大面积插画
-   一堆标签
-   看不清的小字

------------------------------------------------------------------------

# 信息密度

不要把封面做成 PPT。

不要堆砌：

-   高频考点
-   重点章节
-   复习提纲
-   高效提分
-   精准复习

除非它们就是文章主题。

------------------------------------------------------------------------

# 设计理念

每一个元素都必须回答：

> 它是否帮助读者更快理解文章主题？

如果不能：

删除。

坚持：

**Less Decoration. More Information.**

------------------------------------------------------------------------

# 最终检查

-   标题是否足够大？
-   是否只有一个视觉中心？
-   是否符合 Claude Code 风格？
-   是否保持统一品牌？
-   是否一眼知道文章主题？

------------------------------------------------------------------------

# Examples（优秀案例）

> 本章节持续维护。

每当生成一张满意的封面：

-   收录图片
-   记录文章主题
-   总结为什么优秀
-   标注版本号

随着案例增加，不断迭代设计系统。

------------------------------------------------------------------------

# Bad Examples（反面案例）

持续收录：

-   元素过多
-   没有视觉中心
-   标题太小
-   装饰喧宾夺主
-   像广告而不是知识内容
